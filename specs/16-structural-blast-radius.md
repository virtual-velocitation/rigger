# 16 - Structural blast-radius: two contracts, safe by construction

**Goal:** improve the blast radius the conductor keys off - partitioning, gate selection, review tiers - using the symbol index's cross-reference graph (spec 15), WITHOUT making it less safe. Blast-radius has OPPOSITE error costs for its two consumers (architecture 5.5.1): grounding wants precision; partitioning wants recall (a MISSED reference can co-schedule two conflicting units in one parallel batch). So this spec delivers TWO views - a precise one for what an agent reads, and a SAFE-SUPERSET one (`structural ∪ grep`) for what the conductor runs in parallel and routes to review tiers - wires them into the existing `grounded_seed` seam so the tier router finally USES true structural width instead of a fuzzy count, gates the switch behind a partitioning-SAFETY eval, and audits every decision. Adds no new grounding surface and no MCP.

## Design

Implements architecture 5.5.1 (the two contracts), 5.5.5 (the conductor consumer), 5.5.8 (the safety eval), and 5.5.9 (fail-safe + the audit event). Builds on spec 15's per-language, fan-out-scored cross-reference graph; does not rebuild it.

**Unit 1 - the two-view `blast_radius`, safe by construction.** `blast_radius(file|symbol)` returns two views over the cross-reference graph (5.5.1): a PRECISE / ranked view (definers and referencers, ranked) and a SAFE-SUPERSET view (`structural ∪ grep`, uncapped) for the safety consumers. Because the safety view runs BOTH engines - the grep tree-walk AND the index - it is strictly a superset of today's grep radius, so it can never under-partition relative to the current baseline, at the honest cost of more work off the grounding hot path (5.5.9). A hub symbol - one whose reference degree exceeds the repo's degree-distribution percentile (5.5.2) - fails SAFE in the superset by serializing (conflict-with-everything), never truncated. Owns: the two-view query, the safe-superset union, and the serialize-on-hub composition (over-inclusion, never under). Exclusion: the cross-reference graph and its per-repo fan-out scoring are spec 15's; the conductor's consumption is unit 3's; the safety eval that authorizes it is unit 2's.

**Unit 2 - the partitioning + routing safety eval, the go/no-go gate, after unit 1.** Two arms with different jobs (5.5.8). (a) An IMPLEMENTATION-INVARIANT guard: on an adversarial corpus (macro, trait-object, re-export, reflection, common-name) the safety view must be a superset of grep - red only if the union is built wrong (it intersects, or drops the grep side), since recall cannot regress by construction; a regression guard, not a discovery mechanism. (b) The real, QUANTIFIED go/no-go on a PINNED polyglot repo: a parallelism-retention gate (`>= X%` of units stay co-schedulable versus the grep baseline; median safety-radius `<= Y`) AND a tier-routing-distribution check (the light / full split under symbols must not collapse to all-full versus the turbovec / grep baseline). These are the arms that can actually fail. Owns: the partitioning + routing safety validation and its quantified pass/fail gate. Exclusion: the query is unit 1's; the conductor wiring this gate authorizes is unit 3's.

**Unit 3 - wire into `grounded_seed`, audited, gated on unit 2.** With the safety eval green: `grounded_seed` seeds the prompt from the PRECISE view; partitioning and BOTH review-tier signals read the SAFE-SUPERSET view. High-risk-path MEMBERSHIP is tested over the UNCAPPED view (a beyond-cap high-risk file still forces the full panel - the size cap never gates membership), and the tier SIZE signal now USES the true structural width via a `threshold` RE-TUNED on the structural-width distribution (a wide change earns the full panel) rather than the old `<= 8` count. Every unit's radius is recorded on a `BlastRadiusComputed` event - seed files, view, and index content-hash + grammar / tag-query version - emitted on EVERY path including the empty-radius fail-safe (so "why the full panel?" is always answerable); the context-graph projector ignores it idempotently (5.5.9). A runtime wave-level parallelism-retention metric derivable from these events carries a warn threshold so a silently-serializing fleet is visible in production. The empty-radius fail-safe (full panel, unpartitioned) is preserved; with symbols inactive, `grounded_seed` is byte-for-byte unchanged. Owns: the conductor consumption, the structural-width tier re-tuning, the `BlastRadiusComputed` audit event, and the runtime retention metric. Exclusion: the two-view query is unit 1's; the safety gate is unit 2's.

## Global constraints

- Hyphens, not em dashes. New event types: ONE - `BlastRadiusComputed`, the audit record for the quantity that governs partitioning safety, gate depth, and tier routing (5.5.9); it is pure audit and the context-graph projector ignores it idempotently. Everything else rides existing `grounded_seed` / `SpawnOpts.blast_radius`. SAFE BY CONSTRUCTION: the partitioning / routing view is a superset of grep, so it can never under-partition, and a hub symbol serializes rather than truncating - so unit 2's quantified arms prove VALUE (retained parallelism, non-collapsed tier routing), not safety. Gated on spec 15's symbols index; with it inactive `grounded_seed` keeps its current fuzzy top-k behavior UNCHANGED, so both feature lanes are green and the shipped default is unaffected. The empty-blast-radius fail-safe is preserved exactly (empty -> full / unpartitioned, never `light`).
