# 17 - Code-intelligence correctness fixes: safe-recall, hub robustness, index hygiene, audit fidelity

**Goal:** close ten verified correctness and observability defects in the symbol index, the `symbols`/`hybrid` grounders, and the structural blast-radius consumers, WITHOUT changing the shipped default behavior. Each defect below is latent behind opting into `defaults.grounder: symbols`/`hybrid` (and, for one, `partition: by-blast-radius`); the turbovec default is unaffected. The unifying themes: the safe-superset radius must reach EVERY safety consumer (not only wave partitioning), hub detection must not degenerate on realistic distributions, the incremental index must match a fresh build, and the audit/observability surface must reflect reality.

## Design

Builds on the symbol index and cross-reference graph (the `symbols` grounder) and the two-view blast-radius and `BlastRadiusComputed` audit already present. Adds NO new event type and NO new grounding surface. Determinism, feature-gating, and the loud missing-feature error are preserved exactly.

**Unit 1 - symbols grounder + index hygiene (no dependencies).** Three self-contained fixes in `src/grounder/symbols/`.
- (1a) `reindex` of a DELETED or unreadable file must REMOVE that file's symbols from the index. Today the index only gains entries via a successful re-insert, so a file that fails to read leaves its stale defs/refs behind and `ground`/`blast_radius` surface a nonexistent file as a seed - diverging from a fresh whole-tree build that never visits it. Add a removal path (e.g. `SymbolIndex::remove_file`) and call it when a changed file cannot be read or extracted, so the incremental index equals a fresh `build_index` over the same tree.
- (1b) `blast_radius` must apply the SAME empty-terms guard as `ground`. Today `ground` early-returns an empty result when the query yields no symbol terms (e.g. a one-character or all-punctuation query dropped by the length filter), but `blast_radius` falls through and runs an unbounded whole-repo grep, so `precise` is empty while `safe` covers almost the entire tree - forcing the full panel and corrupting the retention metric. Extract ONE shared term-extraction function that both `ground` and `blast_radius` call, and have `blast_radius` return an empty radius (empty precise, empty safe, not-serialize) when terms are empty.
- (1c) the `--language` override must resolve to the SAME grammar auto-detection would pick. Today the override maps a language back through a single canonical extension, which flattens the extension that selects the JSX-aware variant, so `--language ts` on a `.tsx` file parses with the non-JSX grammar and mis-extracts symbols in JSX bodies. Make the registry one source of truth so the override selects the correct (JSX-aware) grammar for a `.tsx` file, matching auto-detect.

**Unit 2 - hub-degree robustness + the eval that guards it (no dependencies).** The hub test decides whether a symbol conflicts-with-everything and serializes.
- (2a) hub detection must not degenerate. The current nearest-rank cutoff `degrees_sorted[floor(N * percentile)]` collapses to 1 on any realistic long-tail distribution (a large tail of single-reference names drags the percentile value down to 1), and the `>=` comparison then flags EVERY referenced symbol as a hub, so every unit serializes and parallelism-retention collapses to near zero. Redefine "hub" as a genuine high-degree outlier: a symbol is a hub only when its reference degree is STRICTLY above the cutoff, and a flat or long-tail-only distribution (no meaningful spread) must yield no hubs. The property to guarantee: on a distribution dominated by degree-one names, the fraction of names flagged as hubs stays small (bounded near the intended top percentile), never approaching all.
- (2b) the partitioning-safety eval's corpus must EXERCISE this. Add a realistic long-tail (hapax-heavy) per-language degree distribution to the eval's pinned inputs and a red-on-regression assertion: a cutoff rule that over-serializes such a distribution (retention collapsing toward zero, or nearly all names flagged as hubs) must FAIL the gate. The existing all-distinct-degree corpus passes without touching this failure mode; the new arm makes the gate actually protect the parallelism-retention it certifies.

**Unit 3 - the safe-superset view for ALL safety consumers, gated on nothing new (touches the conductor).** Wave partitioning already reads the safe-superset (`structural union grep`) view; two other consumers with the same recall requirement were left on the precise (name-level) view.
- (3a) cross-wave staleness (`stale_downstream_units`) must ground on the safe-superset view, so a reference visible only to grep (a macro body, a re-export, a reflection string) still marks a downstream unit stale. On the precise view such a unit is not marked stale, its cached green gate verdicts are reused, and it integrates against changed code - the exact fail-open the safe view exists to prevent.
- (3b) decomposition rule-6 conflict detection must ground on the safe-superset view for the same reason, so two units that truly share a (grep-visible) blast radius are detected as conflicting at decomposition time rather than only being serialized at runtime.
The precise view remains what SEEDS an agent's prompt; only the SAFETY consumers move to the safe view. With symbols inactive the behavior is unchanged.

**Unit 4 - audit fidelity + observability, gated on nothing new.** Four fixes so the `BlastRadiusComputed` audit and the retention metric tell the truth.
- (4a) `Hybrid` must implement `index_stamp` by delegating to its inner structural grounder. Today it inherits the empty-string default, so `record_blast_radius` hits its empty-stamp early-return and NO audit event is ever emitted under `defaults.grounder: hybrid`, and retention is unmeasurable for every hybrid run.
- (4b) the audit must record the ACTUAL seed. Today the prompt is seeded by one derivation (a scored, k-line-truncated pass) while the audit records a different one (a k-file-truncated pass), so past the truncation cap the recorded files diverge from the files that actually seeded the agent. Unify them so the recorded precise files are exactly the files that seeded the prompt.
- (4c) the parallelism-retention metric and its warn threshold must reach an operator. Today the metric is computed but no production path surfaces it - `rigger stats` has no retention row and nothing warns during a run - so a silently-serializing fleet is invisible. Surface it on an operator-visible path (a `rigger stats` row, and/or a logged warning when retention drops below the threshold on a run).
- (4d) the feature-independent grounder resolver must give a `hybrid` its own actionable arm. On a binary built without the symbols feature, `defaults.grounder: hybrid` currently falls to the generic "unknown grounder" error; it must instead report the SAME loud, actionable feature-missing message the `symbols` arm gives (rebuild with the feature, or select a built-in grounder), never a silent degrade.

## Global constraints

- Hyphens, not em dashes (a gate checks the diff; U+2014 fails it). No references to any external tool or project in code, comments, or commit messages.
- NO new event types; NO new grounding surface; NO MCP. Fixes ride existing seams.
- Determinism by construction: anything serialized uses `BTreeMap`/`BTreeSet`/sorted `Vec`, never `HashMap`/`HashSet`; content hashing stays line-ending-normalized.
- BOTH feature lanes must compile and pass: `cargo build`/`cargo test` (default) and `--no-default-features`, plus `cargo fmt --check` and `cargo clippy --all-targets -- -D warnings` in both lanes.
- The shipped default behavior is UNCHANGED: with symbols inactive, `grounded_seed`, partitioning, staleness, and tier routing are byte-for-byte as before; the loud missing-feature error is preserved.
