# 15 - The symbol index and the symbols grounder

**Goal:** add the STRUCTURAL grounding axis (architecture section 5.5): a deterministic, dependency-light symbol index - a projection of the code tree, designed once for its several consumers - and a `symbols` grounder that serves the PRECISE grounding contract (5.5.1) by ranking real symbol matches over incidental text, validated against the TURBOVEC default before it becomes one. This spec delivers the index and the precise contract; the safety contract and the conductor wiring are spec 16.

## Design

Every unit is a slice of the ONE shared model in architecture section 5.5: the data model is 5.5.2, extraction 5.5.3, persistence 5.5.4, the grounder consumers 5.5.5-5.5.6, validation 5.5.8.

**Unit 1 - the symbol data model + tags-based extraction (Rust).** The per-file symbol set (definitions: kind, name, span-as-integers; references: name, span) and the derived name-level cross-reference graph, SCOPED PER LANGUAGE, with high-fan-out-name suppression (5.5.2). No tree-sitter type crosses into the data model - it is parser-free (spans are plain integers, `kind` a rigger enum); tree-sitter lives ONLY in a single extraction function over an INJECTED `(grammar, tag query)` pair (5.5.3), demonstrated on Rust. Owns: the parser-free data model, the per-language cross-reference graph with fan-out suppression, and the injected-grammar extraction pipeline. Exclusion: mapping extensions to grammars is unit 2's; persistence is unit 3's; the grounding query and ranking are unit 4's; the blast-radius contracts and the hub-symbol partitioning degree-cap are spec 16's.

**Unit 2 - the grammar registry and the five languages, after unit 1.** The `extension -> (grammar, tag query)` registry (5.5.3): register Rust, C#, the JavaScript/TypeScript family (`.js/.mjs/.cjs/.jsx` and `.ts/.tsx`), Go, and Python as static, feature-gated grammars; auto-detect by extension; author or verify each grammar's tag query. `--language` is an optional override. Owns: language registration and detection. Exclusion: the extraction pipeline that consumes an injected `(grammar, tag query)` pair is unit 1's; this unit only supplies WHICH grammar a file uses.

**Unit 3 - persistence and incremental freshening, after unit 1.** Persist the index under `.rigger/` per project as a SINGLE native substrate, keyed by a line-ending-normalized per-file content hash; `reindex(files)` re-parses only the changed files after a unit integrates (5.5.4). Determinism is BY CONSTRUCTION: **stable sort keys** (symbols by `(name, span.start)`, files by normalized relative path) and **no serialization from a hash-ordered container** (Rust `HashMap` iteration is per-process randomized), so the index is byte-identical across processes and platforms for the same tree and grammar version. Owns: the on-disk substrate, its construction-deterministic serialization, and incremental freshening. Exclusion: the in-memory model and extraction are unit 1's; the grounder that reads the persisted index is unit 4's.

**Unit 4 - the `symbols` grounder, ranking, and validation-vs-turbovec, after units 1-3.** Implement the `Grounder` port over the index for the PRECISE contract: `ground(query, k)` ranks a name match above a merely-related file, above a reference, above an incidental prose mention (5.5.6); wire `grounder_for("symbols")` and `defaults.grounder: symbols`. Acceptance is empirical AGAINST THE REAL DEFAULT: `rigger replay <run> --against` a symbols config diffs the stats against the TURBOVEC baseline (grep is only a floor), behind a quantified meets-or-exceeds-by-a-stated-margin gate (5.5.8); the shipped default does NOT change until that gate passes. Owns: the grounder consumer, its ranking, config wiring, and the quantified replay validation versus turbovec. Exclusion: `blast_radius` and the safety contract are spec 16's; the vector-composed `hybrid` mode is unit 5's; the index is units 1-3's.

**Unit 5 - the `hybrid` grounder, after unit 4.** A `hybrid` mode composing structure (symbols) with semantics (turbovec): structural symbol matches rank first, the vector pass fills the semantic recall a name match misses. Feature-gated with turbovec; absent turbovec it is exactly the `symbols` mode. Validated by replay against turbovec-alone. Owns: the structure-plus-semantics composition and its validation versus turbovec-alone. Exclusion: the `symbols` mode and its ranking are unit 4's; the turbovec vector engine is the existing turbovec grounder, not re-implemented here; `blast_radius` is spec 16's.

## Global constraints

- Hyphens, not em dashes. New event types: NONE in this spec - the index is a derived, rebuildable `.rigger/` projection; the one new event (`BlastRadiusComputed`) belongs to spec 16, where blast-radius is computed. The domain model is tree-sitter-free (dependency direction, 5.5.2). The grammar set is feature-gated (a `symbols` feature carried in the default build, as `turbovec` is); with the feature off, or for a file whose language is not registered or not parseable, the `symbols` grounder degrades to grep (5.5.9), so BOTH feature lanes compile and pass. The shipped default is UNCHANGED (turbovec / grep stay the default; `symbols` is opt-in) until the quantified replay gate versus turbovec earns the switch. Determinism holds: same tree -> byte-identical index.

## Done when

- [ ] a Rust file's definitions and references (spans as integers) and its per-language-scoped, fan-out-suppressed cross-reference graph are extracted by a parser-free data model fed by a single extraction function over an INJECTED tree-sitter `(grammar, tag query)` - pinned by a fixture with known symbols and a known reference, plus an assertion that no tree-sitter type appears in the data-model API
- [ ] the `extension -> (grammar, tag query)` registry resolves and extracts symbols for all five shipped languages (Rust, C#, JavaScript/TypeScript, Go, Python) with auto-detection by extension and a working `--language` override - pinned with one fixture per language plus an ambiguous-extension override case
- [ ] the index persists as one native substrate under `.rigger/` with construction-deterministic serialization (stable sort keys, no hash-ordered containers) and line-ending-normalized content-hash keying, and `reindex(files)` re-parses only the named files - pinned by a determinism test that RE-INDEXES IN A FRESH PROCESS and asserts a byte-identical index for the same tree + grammar version (catching the hash-seed hazard) AND a single-file-delta test
- [ ] the `symbols` grounder ranks a name match above a related file above a reference above an incidental prose mention, is selectable via `defaults.grounder: symbols`, and a `rigger replay --against` run reports a quantified stats diff versus the TURBOVEC baseline (not grep) that must exceed it by a stated margin, with the shipped default left unchanged until it clears the gate - pinned by a ranking fixture AND the replay-vs-turbovec comparison
- [ ] the `hybrid` grounder ranks structural symbol matches ahead of turbovec's semantic hits, degrades to the `symbols` mode when turbovec is absent, and is measured by replay against turbovec-alone - pinned with turbovec present AND a feature-off control
